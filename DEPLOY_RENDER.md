# Render Deployment Guide

Render is optional. Normal Mako IQ use is local-first through Mako IQ Companion, the local backend at `http://127.0.0.1:8787`, and Kimi/Moonshot from that backend. Use this guide only when you intentionally want a hosted backend.

This repo is prepared so [romanbeck-svg/repos1](https://github.com/romanbeck-svg/repos1) can be the GitHub source of truth for Render deployment.

## Source Of Truth

- GitHub repo: `romanbeck-svg/repos1`
- Default branch: `main`
- Optional staging branch: `staging`
- Deployable backend root: `backend/`
- Active extension source: `src/canvy/**` with `public/manifest.json`

Render should deploy the backend from the `backend/` subdirectory, not from the repo root.

## Render Service Settings

Use these settings when you connect the GitHub repo in Render.

- Repo: `romanbeck-svg/repos1`
- Branch: `main`
- Root Directory: `backend`
- Runtime: `Node`
- Build Command: `npm ci && npm run build`
- Start Command: `npm run start`
- Health Check Path: `/health`

The repo also includes `render.yaml` if you prefer the Blueprint flow. That file expects the same backend root directory and commands.

## Required Render Environment Variables

Enter these in the Render dashboard for the production backend service:

```text
NODE_ENV=production
HOST=0.0.0.0
AI_PROVIDER=kimi
KIMI_BASE_URL=https://api.moonshot.ai/v1
KIMI_MODEL=kimi-k2.6
ALLOW_ALL_EXTENSION_ORIGINS=false
ALLOW_ANONYMOUS_USAGE=true
MOONSHOT_API_KEY=YOUR_REAL_MOONSHOT_KEY
ALLOWED_EXTENSION_ORIGINS=chrome-extension://YOUR_EXTENSION_ID
```

Render can generate this automatically because `render.yaml` already declares it:

```text
JWT_SECRET=<generated by Render>
```

Optional but recommended after the service URL exists:

```text
APP_URL=https://YOUR-RENDER-SERVICE.onrender.com
AUTH_MAGIC_LINK_REDIRECT_URL=https://YOUR-RENDER-SERVICE.onrender.com/auth/callback
```

Optional only if you also have a browser-based web client that should call the backend:

```text
ALLOWED_ORIGINS=https://YOUR_WEB_APP_ORIGIN
```

Do not set `PORT` in Render. Render injects it automatically and the backend already listens on `process.env.PORT` with a local fallback.

## Extension Backend URL

The extension resolves its backend URL in `src/canvy/shared/config.ts`.

You can point the extension at the hosted Render backend in either of these places:

1. Build-time default:
   - file: `.env`
   - key: `VITE_MAKOIQ_API_BASE_URL`
   - value example: `https://YOUR-RENDER-SERVICE.onrender.com`
2. Runtime override:
   - open the extension `Options` page
   - field: `API base URL`
   - value example: `https://YOUR-RENDER-SERVICE.onrender.com`

The runtime override wins if both are set.

## Allowed Extension Origin

The backend allowlist lives in the Render env var:

```text
ALLOWED_EXTENSION_ORIGINS=chrome-extension://YOUR_EXTENSION_ID
```

How to get the value:

1. Build the extension with `npm run build`.
2. Load the unpacked extension from `dist/`.
3. Open `chrome://extensions`.
4. Copy the Mako IQ extension ID.
5. Enter `chrome-extension://<ID>` into `ALLOWED_EXTENSION_ORIGINS` in Render.

If you later change the extension ID or package source, update this Render value to match.

## Rebuild / Reload Rules

- If you change the root `.env` build-time URL, rebuild the extension with `npm run build`.
- After any rebuild, reload the unpacked extension in `chrome://extensions`.
- If you use the Options page runtime override instead, you do not need a rebuild for the backend URL change.

## Health Check Verification

After the Render deploy finishes, open:

```text
https://YOUR-RENDER-SERVICE.onrender.com/health
```

The JSON response should show:

- `ok: true`
- `environment: "production"`
- `analysisConfigured: true`
- `jwtSecretConfigured: true`
- `extensionOriginsConfigured: true`
- `allowedExtensionOriginsCount` greater than `0`

If `analysisConfigured` or `extensionOriginsConfigured` is `false`, the missing Render env vars are the problem, not the code layout.

## Common Failure Points

- No origin or wrong remote:
  The local repo must point at `https://github.com/romanbeck-svg/repos1.git`.
- Wrong branch:
  Render should use `main` unless you intentionally create a staging service from `staging`.
- Wrong root directory:
  Render must use `backend`, not the repo root.
- Missing Render env vars:
  `MOONSHOT_API_KEY` and `ALLOWED_EXTENSION_ORIGINS` are required for the production analyze flow.
- Extension still using localhost:
  The extension still resolves `http://localhost:8787` from root `.env` or a saved Options override.
- CORS / origin mismatch:
  `ALLOWED_EXTENSION_ORIGINS` does not exactly match the real `chrome-extension://<ID>` origin.
- Build artifacts accidentally tracked:
  `.env`, `dist.crx`, and `dist.pem` should remain local-only and should not be committed again.

## Local GitHub Push Checklist

This repo is expected to push from:

- branch `main` as the primary branch
- branch `staging` only if you want a second Render service

After pushing, Render only needs the dashboard setup and the missing external env vars.
